# OMM 日报系统 Apple-Inspired UI 设计系统

版本：Apple-inspired UI 分阶段实施记录；当前程序版本为 5.5.2

## 设计目标

OMM 日报系统是生产工具，不做营销页、不做展示型大留白。新的界面方向是 Apple-inspired 的本地桌面工具：清晰、安静、轻盈、层级明确，保留高效扫视和重复操作能力。

## 设计原则

1. 功能优先  
   生成日报、队列审核、手量补录、设置管理仍是第一优先级。视觉只服务识别速度和操作信心。

2. 安静的层级  
   默认界面用中性色和浅色层级；蓝色只用于主操作、当前状态、可点击焦点；红色和琥珀只用于错误与风险。

3. 现代桌面感  
   使用系统字体、柔和背景、细边框、轻阴影和短过渡。避免大面积高饱和渐变、装饰图形和营销式 hero。

4. 收纳复杂度  
   主界面只放日常生成工作流。诊断、日志、本地数据、模板路径、个人清理等放入设置或关于，不污染工作台。

5. 可回滚演进  
   UI 重构分阶段执行：先 tokens 和基础组件，再关键入口，再弹窗与复杂表单。每阶段独立验证和提交。

## 视觉语言

### 色彩

- 应用背景：`#f5f5f7`，接近 macOS/iOS 设置页的低噪声灰。
- 页面容器：`rgba(255,255,255,0.78)`，用于主要面板。
- 普通分隔：`rgba(15,23,42,0.08)`。
- 主色：Apple-like blue `#0a84ff`。
- 主色 hover：`#0071e3`。
- 成功：`#34c759`。
- 警告：`#ff9f0a`。
- 错误：`#ff3b30`。
- 文字主色：`#1d1d1f`。
- 文字次色：`#6e6e73`。

### 字体

优先系统字体栈：

```css
-apple-system, BlinkMacSystemFont, "SF Pro Text", "Segoe UI", "Microsoft YaHei UI", "PingFang SC", sans-serif
```

数字、路径、日志使用等宽系统字体栈。

### 圆角和阴影

- 常规控件：10px。
- 面板和弹窗：14px。
- 重复列表项：10px。
- 阴影：低透明度、短距离，避免厚重卡片感。

### 动效

- 默认过渡：160ms ease-out。
- 弹窗/hover/焦点：轻微透明度、背景和阴影变化。
- 不做复杂转场，不引入业务无关动画。

## 布局策略

### 登录页

- 背景使用柔和系统灰和微弱光感，不放营销文案。
- 左侧为账户操作，右侧为账户列表和数据位置。
- 错误/提示采用统一 strip 样式。

### 主界面

- 顶部为紧凑工具栏：应用名、当前账户、设置、帮助。
- 主体保留工作流密度：目录、日期文件夹、队列、生成动作。
- 移除开发味日志和诊断显示；诊断仅在设置 > 关于软件里打开。
- 队列条目要更像 Finder/Numbers 的列表项，边框轻、状态标签清楚。

### 设置中心

- 左侧导航采用 macOS 设置式 sidebar。
- 右侧内容使用 grouped sections。
- “关于软件”承载版本、本地数据管理、诊断日志。
- 个人清理中心采用分类导航、项目列表和详情面板三段式布局；清理项必须同时说明“会清理、会保留、可能影响、备份策略”，避免危险开关堆叠成无序列表。
- 设置中心外壳固定高度并隐藏外层滚动，只允许右侧内容区滚动；在 700px 左右窗口宽度仍保持左侧导航，避免顶部双列分类和弹窗外层滚动条同时出现。

### 帮助中心

- 保留搜索。
- 左侧分类和中间主题列表采用轻边框、浅灰底、白色选中态。
- 正文保持可读行高，不追求花哨。

### 弹窗

- 所有弹窗统一 overlay、圆角、标题区、底部按钮区。
- 危险弹窗保留明显警告色。
- 手量/审核类弹窗保持信息密度，不改业务流程。

## 组件规范

### Button

- default：蓝色主按钮，白字。
- outline：白底细边框，hover 为浅灰。
- ghost：透明按钮，hover 为浅灰。
- destructive：红色按钮，仅用于危险动作。
- 高度：small 32px，default 38px，large 44px，icon 36px。

### Input / Select

- 白色或半透明白底。
- 细边框，focus 使用蓝色 ring。
- 不使用厚重阴影。

### Card / Panel

- 页面面板可用白色半透明和细边框。
- 不嵌套装饰卡片；重复项可以用卡片。
- 设置区块使用 section，不堆叠多层浮卡。

### Status Strip

- info：蓝色低饱和。
- warning：琥珀低饱和。
- error：红色低饱和。
- success：绿色低饱和。

## 第一阶段实施范围

1. 全局 CSS tokens：颜色、字体、背景、滚动条、通用 strip/panel/pill。
2. 基础组件：Button、Card、Dialog、Input。
3. 登录入口：背景、容器、账号列表、提示区。
4. 主窗口框架：背景、顶部栏、基础面板质感。
5. 设置中心：sidebar、内容背景、关于软件数据管理区。

## 第二阶段实施范围

1. 统一预览、审核、手量补录、个人清理、识别补充、模板位置和生成结果等复杂业务弹窗。
2. 弹窗统一使用轻量遮罩、白色半透明面板、细边框、紧凑信息块和更清晰的结果/警告容器。
3. 本阶段只调整视觉层，不改变生成、识别、保存、清理等业务逻辑。

## 第三阶段实施范围

1. 统一主界面待生成队列，使用轻量列表容器、圆角行、选中态和状态 pill。
2. 统一右键菜单为半透明浮层、圆角菜单项和清晰的危险/警告操作色。
3. 统一设置中心内部信息块、入口卡片和关于/本地数据管理卡片。
4. 本阶段仍只调整视觉层，不改变队列、单日设置、账户、配置和数据管理逻辑。

## 第四阶段实施范围

1. 统一帮助中心二级列表、搜索结果和空结果提示。
2. 统一选择班次、单日设置等小弹窗外壳和内部信息块。
3. 将每件时间偏低确认、个人清理危险确认从系统原生确认框改为应用内确认浮层。
4. 本阶段仍不改变识别、排程、生成和清理执行逻辑。

## 第五阶段实施范围

1. 精修登录页、特殊大件、识别补充、审核、预览、帮助页脚、通用弹窗页脚和关闭按钮的剩余旧样式。
2. 统一剩余表格和列表小块为白色半透明、轻描边、短阴影和蓝色焦点反馈。
3. 设置中心禁用输入、关于页和工具入口继续收敛到同一视觉语言。
4. 本阶段仍只调整 UI 表现，不改变排程、识别、生成、账号、清理和本地数据逻辑。

## 5.3.4 预览页简化

1. 预览页顶部不再一次性展示所有时间字段，只保留“可生成/需处理”的结论、有效计入、最低要求和差额。
2. 总在岗、纯工作、固定休息、缓冲余量、目标/实际下班和来源拆分默认收纳到“查看计算细项”。
3. “当天合格”状态改为用户能读懂的结论说明，减少名词堆叠，并在细项中补充解释。
4. 本阶段仍只调整展示层，不改变排程和生成结果。

## 5.5.2 收尾审查

1. 主操作区居中前置，预览日报和生成报表脱离左侧配置区，减少主流程误读。
2. 设置中心、手量补录、个人清理中心和识别补充规则统一为固定弹窗外壳，头尾固定，内容区单独滚动。
3. 通用 Dialog 外层不再自带滚动，避免外层和内容区双滚动条贴边。
4. 登录页机构水印、启动页、浏览器标题和 dev 入口版本同步到 5.5.2。
5. 本阶段仍只调整 UI 表现，不改变排程、识别、生成、账号、清理和本地数据逻辑。

## 暂缓范围

- 不改 sidecar、排程、识别、生成算法。
- 不改数据库结构，除非后续数据管理继续扩展。
- 不重写所有弹窗内部逻辑。
- 不引入外部 UI 框架。
- 不打包，只做 dev/build/smoke 验证。

## 验收清单

- 主界面没有开发日志常驻。
- 登录、设置、帮助、主工作台视觉语言一致。
- 所有按钮和输入框有清楚 hover/focus/disabled。
- 文本在窄窗口不溢出主要容器。
- `npx.cmd tsc --noEmit` 通过。
- `cargo check --release` 通过。
- `npm.cmd run smoke` 通过。
