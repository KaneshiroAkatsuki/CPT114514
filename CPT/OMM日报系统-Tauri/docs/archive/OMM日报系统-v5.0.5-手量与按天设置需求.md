# OMM日报系统 v5.0.5 需求交接：手量补录与按天单独设置

> 本文档用于接力开发。当前 v5.0.4 已完成便携版模板兜底、下班策略、隐形缓冲、队列项 settingsOverride 等基础修复。v5.0.5 的重点是把“真实手量”和“按日期条目单独设置”做成可用功能。

## 1. 已确认业务规则

### 1.1 班次与休息

- A 班/白班开始：08:00。
- B 班/夜班开始：20:00。
- 固定休息总计 2 小时，A/B 班只是具体时间不同。

A 班正常班休息：

- 10:00-10:10
- 11:40-12:40
- 15:10-15:20
- 17:00-17:40

B 班正常班休息：

- 22:00-22:10
- 23:20-00:20
- 03:10-03:20
- 05:40-06:20

下早班只使用前三段休息。

### 1.2 下班策略

下班策略有三种：

- `early`：下早班。
- `normal`：不下早班。
- `auto`：智能判断，可下早班也可不下早班。

下早班：

- 实际工作日为 8 小时，不包含固定休息。
- 报表有效工作时间必须满 7.5 小时，即 450 分钟。
- A 班目标下班 17:20，B 班目标下班 05:20。
- 最终完成时间尽量控制在下班前 10-15 分钟，即 A 班约 17:05-17:10，B 班约 05:05-05:10。

不下早班：

- 实际工作日为 10 小时，不包含固定休息。
- 报表有效工作时间必须满 9.5 小时，即 570 分钟。
- A 班目标下班 20:00，B 班目标下班 08:00。
- 最终完成时间尽量控制在下班前 10-15 分钟，即 A 班约 19:45-19:50，B 班约 07:45-07:50。

`auto` 策略：

- 先尝试 `early`。
- 如果排程后能在 early 目标下班时刻前完成，就选择 `early`。
- 如果 early 会超时，则回退 `normal`。
- 如果 normal 也超时，则尝试压缩到每件最低 2.5 分钟。
- 如果最低 2.5 分钟/件仍超时，必须给出醒目警告，建议用户把部分包移到下一天或手动调整。

注意：`auto` 不应只看“自然任务是否超过 450 分钟”，而应看真实排程结果是否能在目标时刻前完成。

### 1.3 隐形缓冲

- 隐形缓冲用于补足报表有效工作时间，让最终时间尽量靠近下班前 10-15 分钟。
- 隐形缓冲算入有效工作时间。
- 隐形缓冲不写入最终 Excel。
- 预览窗口可以显示“隐形缓冲”行，用琥珀色或弱化样式标记。
- 每段隐形缓冲最长 10 分钟。
- 只有当前结束时间早于理想最小结束时间时才插入隐形缓冲。

## 2. 按日期条目单独设置

当前界面已有全局设置，但实际使用中每个日期可能不同。例如某一天真的下早班，另一天下正常班；某一天真的有手量，另一天天没有。

### 2.1 目标

队列中每个日期条目都应支持单独覆盖设置。全局设置作为默认值，单个日期条目可以覆盖部分字段。

建议字段：

- 下班策略：继承全局 / 智能判断 / 下早班 / 不下早班。
- 是否启用手量：继承全局 / 是 / 否。
- 是否启用其他事务：继承全局 / 是 / 否。
- TPP 范围：继承全局或单独设置。
- 包间休息：继承全局或单独设置。
- 手量上限：继承全局或单独设置。
- 其他事务上限：继承全局或单独设置。
- 特殊物品规则：继承全局或单独设置。
- 真实手量任务：该日期专属的手量记录列表。

### 2.2 UI 建议

优先使用队列项右键菜单和“单日设置”弹窗：

- 右键日期条目，选择“单日设置”。
- 弹窗顶部显示日期文件夹名、识别到的班次、当前生效策略。
- 弹窗中分区显示：
  - 下班策略
  - 手量/其他事务
  - TPP 与休息
  - 真实手量任务
  - 预览影响摘要

### 2.3 持久化

当前不强制要求持久化到配置文件。

建议先让 `settingsOverride` 只在当前运行期间生效。原因：

- 单日设置通常只对应本次处理的日期。
- 默认跨天恢复容易误用旧设置。
- 后续如有需要，再做“保存本批次配置/导入配置”，不要默认永久记住所有覆盖项。

## 3. 真实手量功能

之前“手量”主要用于补时间，现在新增“真实手量”概念。

### 3.1 真实手量需要写入报表的信息

如果用户确认某天确实有手量，则需要补充并写入报表：

- 工站：固定为 `手量`。
- 品名：用户填写或从文件夹名识别。
- 送测人：用户填写或从文件夹名识别。
- 工单号：没有则 `/`。
- 模号：没有则 `/`。
- 机台号：没有则 `/`。
- 首件/制程/测试尺寸：一般为 `测试尺寸`。
- 送测项目：固定为 `OMM`。
- 送测日期：用户填写或从文件夹名识别。
- 送测时间：用户填写，缺失可用 `/`。
- 测试数量：用户填写。
- 测试日期：一般为当天日期。
- 开始时间/完成时间/耗时：由用户填写的手量耗时参与排程后生成。
- 状态：已完成。
- 测量员：从文件夹名识别或用户填写。

参考截图中第 12、13 行：

- 工站为 `手量`。
- 品名为 `565`。
- 送测人为 `安睿克`。
- 送测项目为 `OMM`。
- 送测日期可能为 `6月28日`。
- 测试数量为 `96PCS`。
- 测量员为 `卫阳`。

### 3.2 手量文件夹命名识别

手量文件夹常见特征：

- 文件夹末尾或接近末尾出现 `-手量-禹欣`，表示这是手量任务，最后名字是测量员。
- 可能与 `-OMM-名字`、`-CMM-名字` 放在一起或夹在中间。
- 可能出现用户名字是 `禹欣`，但文件夹里有 `-OMM-王业陈`。这代表 OMM 由别人测量，禹欣可能负责手量。

识别建议：

- 使用 `-手量-姓名` 作为强特征。
- 不要把 `-OMM-姓名` 简单等同于手量测量员。
- 如果同时出现 `手量`、`OMM`、`CMM`，优先识别手量段落，并把识别结果放入手量补录窗口让用户确认。
- 识别不完整时不要静默生成，必须提示用户补齐。

### 3.3 手量任务管理窗口

这不是普通报错弹窗，而应是一个独立功能窗口。

建议名称：

- “手量任务管理”
- 或“手量补录”

核心能力：

- 支持粘贴一个或多个文件夹名，自动识别手量信息。
- 支持手动新增手量记录。
- 支持编辑识别结果。
- 支持为当前日期绑定手量记录。
- 支持删除某条手量记录。
- 支持将识别不完整的字段高亮。

建议布局：

- 左侧：粘贴识别区。
  - 多行文本框：每行一个文件夹名。
  - 按钮：识别手量。
  - 显示识别日志或无法识别项。
- 右侧：手量记录表格。
  - 品名
  - 送测人
  - 送测日期
  - 送测时间
  - 测试数量
  - 测试耗时
  - 测量员
  - 备注/来源
- 底部：
  - 添加记录
  - 删除记录
  - 保存到当前日期
  - 保存并预览
  - 取消

### 3.4 手量耗时输入

真实手量耗时由用户填写，参与排程。

支持格式：

- `1.5H`
- `1.5h`
- `1小时30分钟`
- `1小时`
- `90分钟`
- `1:30`
- `01:30`

规则：

- 单位内部统一为分钟。
- 正常上限 180 分钟，即 3 小时。
- 超过 180 分钟时不直接禁止，但应弹出确认或醒目警告。
- 低于 5 分钟时应提示不合理。
- 手量耗时不受 OMM 每件 2.5 分钟最低规则限制，因为这是独立手量任务。

### 3.5 真实手量与“补时间手量”的区别

必须区分两类：

1. 真实手量：
   - 用户确认存在。
   - 有完整报表字段。
   - 写入最终 Excel。
   - 参与排程。

2. 补时间手量/其他事务：
   - 用于补足有效工作时间。
   - 可以保持当前功能。
   - 如果没有完整字段，不应伪装成真实手量。

建议数据结构中明确标记：

- `manual_kind: "real"`：真实手量。
- `manual_kind: "filler"`：补时间手量。
- 或使用独立字段 `is_real_manual: true/false`。

## 4. 预览与生成规则

### 4.1 预览时没有手量信息怎么办

预览时可能还没填写真实手量信息。此时不要假装已经识别到手量。

建议处理：

- 如果没有疑似手量，也没有用户勾选“确实有手量”：
  - 正常预览。
- 如果识别到疑似手量文件夹，但信息不完整：
  - 预览窗口顶部显示提示：“发现疑似手量，请补录后再生成正式报表。”
  - 提供按钮：“去补录手量”“忽略并预览”“仅按隐形缓冲预览”。
- 如果用户勾选“确实有手量”，但没有补齐必要字段：
  - 允许预览，但必须标记为“手量信息未完整，正式生成前需补齐”。
  - 正式生成时应阻止或二次确认。

### 4.2 正式生成

正式生成时：

- 真实手量字段完整：写入 Excel。
- 真实手量字段缺失：阻止生成或要求用户确认忽略真实手量。
- 隐形缓冲：不写入 Excel。
- 固定休息：不写入 Excel。
- 包间休息：按现有规则处理。
- `sched_warnings`：生成完成弹窗必须有醒目的黄色警告块。

## 5. 最小实现建议

不要大规模重构。建议分阶段做：

### P0：手量耗时解析与数据结构

- 新增解析函数：`parse_manual_duration(input: string) -> minutes`。
- 新增真实手量记录类型。
- 新增字段校验函数。

### P1：手量补录窗口

- 前端新增 `ManualTaskDialog` 或类似组件。
- 支持粘贴文件夹名识别。
- 支持手动编辑字段。
- 将记录保存到当前队列项 `settingsOverride.real_manual_tasks`。

### P2：后端排程接入

- `preview/generate` 接收 `real_manual_tasks`。
- 将真实手量转换为排程 segment。
- 真实手量写入 Excel。
- 补时间手量继续保持旧逻辑。

### P3：预览与警告

- 预览显示真实手量行。
- 预览显示疑似手量未补齐警告。
- 生成完成弹窗显示 `sched_warnings` 和手量校验警告。

## 6. 验证要求

测试目录：

- 原始数据：`C:\Users\Administrator\Desktop\勿动\日期文件`
- 禁止移动、删除、重命名原始数据。
- 测试输出：`D:\KSoftware\KMAA\CPT\OMM日报系统-Tauri\test-output`
- 不要写回原始测试数据目录。

至少验证：

- 无手量时：旧流程不受影响。
- 有疑似手量文件夹时：能识别并进入补录。
- 手量信息完整时：生成 Excel 中出现真实手量行。
- 手量耗时 `1.5H`、`1小时30分钟`、`90分钟`、`1:30` 都能解析为 90 分钟。
- 超过 3 小时有警告。
- 下早班目标仍为 450 分钟，正常班仍为 570 分钟。
- 隐形缓冲仍不写入 Excel。
- 任务过多仍返回警告。

## 7. 当前暂不做

- 暂不要求把每个队列项的 `settingsOverride` 永久保存到配置文件。
- 暂不要求把 auto 阈值做成用户可配置。
- 暂不要求一次性识别所有复杂命名，只要强特征 `-手量-姓名` 稳定即可。
- 暂不改动原始测试数据目录。
