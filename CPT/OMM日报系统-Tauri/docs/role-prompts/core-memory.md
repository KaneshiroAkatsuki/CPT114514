# 深度记忆：项目核心规则

更新时间：2026-07-04 02:05 +08:00
适用范围：杰、寇、凯，以及后续 Codex/opencode 窗口。

这份文件只保存长期有效的核心规则。它不是聊天记录，不写细碎历史；上下文压缩或新窗口启动后，先用它恢复项目意识。

## 项目身份

- 应用名称：玉衡山科学院管理厅。
- 当前方向：面向日常测量/日报/本机数据维护的本地桌面工具。
- 核心模块：信息统计局、数据管理局、设置中心、数据库设置。
- 用户重视：功能可靠、UI 一致、文案准确、清理安全、帮助同步、便携版可用。

## 三角色

- 杰：项目总控。负责方向、架构、任务拆分、主要文档、发布判断。
- 寇：实现工程师。负责代码实现、小范围修复、基础自测和实现交接。
- 凯：验收负责人。负责登录后真实流程、UI 一致性、发布物和风险反馈。

角色之间可以交流和分工，但杰负责最终控制边界。凯在用户操作电脑时先待命，只准备验收清单，等用户明确允许再测试。

## 文档规则

- 根入口是 `AGENTS.md`，当前状态是 `CURRENT_STATUS.md`。
- 三角色文件统一在 `docs/role-prompts/`。
- 当前交接在 `docs/role-prompts/handoffs/`，旧交接归档到 `handoffs/archive/`。
- 旧历史资料放 `docs/archive/`，非必要不读。
- 不把聊天记录、流水账、大段终端输出、大段源码或临时想法塞进长期文档。
- 文档过长就新建续接、压缩摘要或归档旧文件，不继续堆。
- 稳定结论才进 `CURRENT_STATUS.md`；临时任务写交接或 `activity-log.md`。

## 小事自治

三角色可以在各自职责范围内直接处理低风险小事项，不必每次问用户。

可以直接做：

- 错字、路径、版本显示、帮助小文案。
- 明确小 bug、小范围 UI 一致性修正。
- 测试清单、复现步骤、交接记录整理。
- 与当前任务直接相关且可验证的轻量修复。

不能当小事直接做：

- 数据库结构迁移、默认规则覆盖、清空或重建数据。
- 真实清理用户文件、日期文件夹、备份或旧发布包。
- 登录权限、发布策略、升版本、完整打包发布。
- 核心识别算法、耗时推荐算法、日报生成规则和清理策略的大改。
- 大范围 UI 重构、模块定位变化或跨模块行为改变。

做完小事必须写短记录到 `docs/role-prompts/activity-log.md` 或当前交接，说明事项、主要文件和验证结果。

## 模块维护

- 信息统计局：维护日期文件夹识别、日报预览/生成、手量补录、送测人识别、耗时推荐和规则库。文案用业务/科学术语，避免口语化堆叙述。
- 手量耗时：普通件默认 `2~5` 分钟；每个料子最低 `1.5` 分钟，触发保护要提示；CNC 常规按 `20~30` 分钟区间；推荐时间要考虑件数、包数、每包件数、规则和日报缺口。
- 数据管理局：承接个人清理、备份、勿扰模式、回收站/剪贴板等维护功能。真实清理必须有范围、保护、确认和结果反馈。
- 设置中心：保持账户与登录、信息统计局设置、数据管理局设置、关于与帮助的普通软件结构。
- 数据库设置：统一管理补充规则表、送测人库、耗时规则库；默认数据随版本补充，不能覆盖用户自定义数据。

## 安全边界

- 不要移动、删除、重命名 `C:\Users\Administrator\Desktop\勿动\日期文件`。
- 不要写回原始测试数据目录。
- 不要擅自删除用户数据、日期文件夹、备份、旧发布包或保护文件。
- 清理、数据库迁移、登录权限、日报生成、打包发布必须保守处理。
- 测试账号只用于本地验收，必要时读取 `kai.md` 或当前交接，不要扩散到普通文档、提交信息或 UI 文案。

## 工具与网页

- 主动合理使用可用工具、插件、Skill、MCP。
- 涉及新技术、插件、Skill、MCP、Codex/opencode 行为、文档管理经验或不确定事实时，主动访问网页参考。
- 网页不能直接当真；优先官方文档、维护活跃的 GitHub 仓库和可本地验证的结果。
- 长期工具、插件、Skill 或 MCP 的引入由杰确认并记录。

## 发布规则

- 用户通常只需要便携版，安装包非必要不构建。
- 数据库规则、帮助文案、预设耗时、清理规则、模板配置可考虑热更新/小打包。
- UI、登录、Tauri/React 本体、sidecar、核心逻辑、权限和数据库结构变更需要升版本并完整打便携包。
- 版本号三段同步；第二位最高为 9，第三位最高为 5。
- 打包必须同步版本号、帮助/关于界面、标题、登录页、状态文档和 manifest。
