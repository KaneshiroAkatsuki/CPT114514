## 2. 发现的问题

### P1：真实手量跨休息拆分后，Excel 重复显示完整测试数量

复现条件：

- B 班。
- 一条真实手量耗时 90 分钟。
- 排程跨过 B 班 05:40-06:20 固定休息。

实际 Excel：

- 同一条真实手量被拆成两行：
  - 04:20-05:40
  - 06:20-06:30
- 两行都显示 `96PCS`。

风险：

- 报表看起来像测了两次 `96PCS`，容易误解为数量翻倍。

建议最小修复：

- 保留“跨休息拆分”的排程行为。
- 给真实手量任务增加稳定 id，例如 `manual_task_id`。
- 当同一条真实手量被拆成多段写入 Excel 时：
  - 第一段保留完整字段和 `96PCS`。
  - 后续段标记为续行，避免重复数量。

建议后续段写法：

- 工站：`手量`
- 品名：同原任务或 `续`
- 送测人：同原任务或 `/`
- 测试数量：建议写 `/` 或 `续`
- 备注：`真实手量续行`
- 开始/结束时间照常写入。

不要把后续段的数量继续写成完整 `96PCS`。

### P1：只有真实手量、没有 OMM 普通记录时，预览失败

复现条件：

- `records=[]`
- `settings.real_manual_tasks` 有一条完整真实手量。

实际结果：

```json
{"success": false, "data": null, "error": "预览失败，请检查日志"}
```

风险：

- 用户已经明确说过：手量文件夹可能不在同一台电脑，可能需要单独补录。
- 某一天如果只有手量补录、没有识别到 OMM 文件夹，当前无法预览/生成。

建议最小修复：

- `preview()` 和 `generate` 流程不要在普通 `records` 为空时直接失败。
- 如果 `records` 为空但 `real_manual_tasks` 非空，应允许继续排程。
- `folder_name`、`shift_label`、`test_date` 仍可从 `base_dir` 文件夹名推导。
- 如果既没有普通记录，也没有真实手量，则仍返回失败。

### P2：生成前没有强制校验真实手量字段完整

当前行为：

- 前端弹窗提示“存在未补齐的真实手量字段，正式生成前请补全”。
- 但保存和生成没有阻止。

建议：

- 正式生成前必须校验 `real_manual_tasks`。
- 必填字段缺失时，不要生成，弹出提示用户补齐。
- 超过 180 分钟可以不禁止，但需要醒目二次确认或 warning。

必填字段：

- 品名
- 送测人
- 送测日期
- 测试数量
- 测试耗时
- 测量员

### P3：测试命令中的 `< file` 不适合 PowerShell

交接文档里的命令：

```powershell
binaries\generate_report.exe < xxx.jsonl
```

在 PowerShell 中会报：

```text
The '<' operator is reserved for future use.
```

建议文档和验证命令改为：

```powershell
Get-Content -Encoding UTF8 xxx.jsonl | .\binaries\generate_report.exe
```

或者用 Python `subprocess.run(input=...)`。
